3) Branching & Merging (Recommended)

If working with partners, each person should contribute at least one commit.

git checkout -b feature/your-name-section
# edit files...
git add .
git commit -m "add my section"
git push -u origin feature/your-name-section

# Open a Pull Request on GitHub and merge it.
# OR, locally:
git checkout main
git pull
git merge feature/your-name-section
git push

4) Capture Your Git History (REQUIRED)

Run one of the following and take a screenshot of your terminal output. Make sure your name and recent commits are visible.

# Clean, compact history (recommended)
git log --oneline --graph --decorate --all --date=short --pretty=format:"%h %ad %an %d %s"

# Or include files changed
git log --date=short --pretty=format:"%h %ad %an %s" --name-status -n 20

# Verify authorship of your commits
git log --author="Your Name" --oneline -n 10

Screenshot 2 — Optional extra

How to take a screenshot

  • Windows: Win+Shift+S → drag to select → paste/save from the Snipping Tool.
  • macOS: Shift+Cmd+4 → drag to select → file saves to desktop.

5) Short Reflection

Quick Fixes (Common Issues)

# Set default branch name (if needed)
git config --global init.defaultBranch main

# Link local repo to GitHub if 'origin' is missing
git remote add origin <repo-url>

# First push and set upstream
git push -u origin main

# Pull the latest before you merge
git pull --rebase

# Stash local changes temporarily
git stash
git pull
git stash pop
